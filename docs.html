Getting Started

Welcome to Selenium documentation! This guide introduces Selenium, why you might use it, and how to set up your environment with Java and Maven/Gradle. You'll also run your first WebDriver script.

Introduction to Selenium

Selenium is a suite of tools for automating browsers. It lets you simulate real user actions‚Äîopening pages, clicking, typing, and verifying results‚Äîacross different browsers.

Why Selenium? (Use cases)

  • End‚Äëto‚Äëend UI test automation for web apps.
  • Cross‚Äëbrowser verification (Chrome, Firefox, Edge).
  • Regression testing and smoke/sanity suites.
  • Supports multiple languages (Java, Python, JS, C#, etc.).

Installation & Setup

Java Setup

Install JDK 17+ and verify:

bash 
java -version
javac -version

Maven/Gradle Setup

Add Selenium Java dependency to your build tool.

xml 

<dependencies>
  <dependency>
    <groupId>org.seleniumhq.selenium</groupId>
    <artifactId>selenium-java</artifactId>
    <version>4.22.0</version>
  </dependency>
</dependencies>
gradle 
// build.gradle
dependencies {
  testImplementation 'org.seleniumhq.selenium:selenium-java:4.22.0'
}

Selenium WebDriver Installation

Selenium 4 uses the built‚Äëin WebDriver BiDi/DevTools where supported. You still need a matching browser driver (e.g., chromedriver) in your PATH or manage it via tools like WebDriverManager.

Your First Selenium Script

java 
import org.openqa.selenium.By;
import org.openqa.selenium.WebDriver;
import org.openqa.selenium.WebElement;
import org.openqa.selenium.chrome.ChromeDriver;

public class FirstTest {
  public static void main(String[] args) {
    // If chromedriver is not on PATH, set:
    // System.setProperty("webdriver.chrome.driver", "/path/to/chromedriver");
    WebDriver driver = new ChromeDriver();
    try {
      driver.get("https://example.org");
      WebElement heading = driver.findElement(By.cssSelector("h1"));
      System.out.println("Heading: " + heading.getText());
    } finally {
      driver.quit();
    }
  }
}

Core Concepts

Learn the pillars of Selenium: WebDriver, element location strategies, element interactions, waits, navigation, windows/tabs, alerts, and frames.

Understanding WebDriver

WebDriver is the API for controlling the browser. You create a driver instance (e.g., ChromeDriver) and issue commands (navigate, find, click...).

Browser Drivers (Chrome, Firefox, Edge...)

  • Chrome: ChromeDriver
  • Firefox: FirefoxDriver
  • Edge: EdgeDriver

Locating Elements

By ID, Name, Class, CSS Selector, XPath

java 
driver.findElement(By.id("email"));
driver.findElement(By.name("q"));
driver.findElement(By.className("btn-primary"));
driver.findElement(By.cssSelector("#login .btn.primary"));
driver.findElement(By.xpath("//input[@type='text' and contains(@class,'search')]"));

Advanced XPath Tricks

java 
// Following sibling
By.xpath("//label[text()='Email']/following-sibling::input");
// Parent navigation
By.xpath("//*[@data-test='price']/parent::div");
// Indexing within groups
By.xpath("(//button[contains(.,'Add')])[2]");

Waits

Implicit Wait

java 
driver.manage().timeouts().implicitlyWait(java.time.Duration.ofSeconds(5));

Explicit Wait

java 
import org.openqa.selenium.support.ui.WebDriverWait;
import org.openqa.selenium.support.ui.ExpectedConditions;

WebDriverWait wait = new WebDriverWait(driver, java.time.Duration.ofSeconds(10));
WebElement btn = wait.until(ExpectedConditions.elementToBeClickable(By.id("save")));
btn.click();

Fluent Wait

java 
import org.openqa.selenium.support.ui.FluentWait;

new FluentWait<WebDriver>(driver)
  .withTimeout(java.time.Duration.ofSeconds(15))
  .pollingEvery(java.time.Duration.ofMillis(500))
  .ignoring(org.openqa.selenium.NoSuchElementException.class)
  .until(d -> d.findElement(By.id("status")).isDisplayed());

Advanced Usage

Go beyond basics: actions, JavaScript execution, screenshots, cookies, dropdowns, tables, file upload/download, and headless runs.

Actions Class (Mouse & Keyboard)

java 
import org.openqa.selenium.interactions.Actions;

Actions actions = new Actions(driver);
actions.moveToElement(driver.findElement(By.id("menu")))
       .click()
       .build()
       .perform();

JavaScriptExecutor (Scroll, Click, Highlight)

java 
import org.openqa.selenium.JavascriptExecutor;

JavascriptExecutor js = (JavascriptExecutor) driver;
js.executeScript("window.scrollTo(0, document.body.scrollHeight)");
js.executeScript("arguments[0].click();", driver.findElement(By.id("save")));
js.executeScript("arguments[0].style.border='2px solid red'", driver.findElement(By.id("email")));

Introduction to Test Automation Frameworks

Popular Java choices: TestNG, JUnit, Cucumber (BDD). Each provides structure, assertions, and reporting.

TestNG Integration

xml 

<dependency>
  <groupId>org.testng</groupId>
  <artifactId>testng</artifactId>
  <version>7.10.2</version>
  <scope>test</scope>
</dependency>
java 
import org.testng.annotations.*;

public class LoginTest {
  WebDriver driver;

  @BeforeMethod
  public void setup() { driver = new ChromeDriver(); }

  @AfterMethod
  public void tearDown() { if (driver != null) driver.quit(); }

  @Test
  public void userCanLogin() {
    driver.get("https://example.org/login");
    driver.findElement(By.id("email")).sendKeys("user@example.org");
    driver.findElement(By.id("password")).sendKeys("secret");
    driver.findElement(By.cssSelector("button[type='submit']")).click();
  }
}

Page Object Model (POM)

java 
public class LoginPage {
  private WebDriver driver;
  private By email = By.id("email");
  private By pass = By.id("password");
  private By submit = By.cssSelector("button[type='submit']");

  public LoginPage(WebDriver driver) { this.driver = driver; }

  public LoginPage enterEmail(String e) { driver.findElement(email).sendKeys(e); return this; }
  public LoginPage enterPassword(String p) { driver.findElement(pass).sendKeys(p); return this; }
  public void submit() { driver.findElement(submit).click(); }
}

Best Practices

Write maintainable, stable tests with clear structure, reusable utilities, and consistent naming.

Writing Maintainable Tests

  • Use POM to encapsulate selectors and flows.
  • Keep tests atomic and independent.
  • Avoid brittle locators; prefer IDs and stable attributes.
  • Centralize waits and common actions.

Reusable Methods & Utilities

java 
public class Commons {
  private WebDriver driver;
  public Commons(WebDriver d){ this.driver = d; }
  public void click(By by){ new WebDriverWait(driver, java.time.Duration.ofSeconds(10))
      .until(ExpectedConditions.elementToBeClickable(by)).click(); }
  public void type(By by, String text){ driver.findElement(by).clear(); driver.findElement(by).sendKeys(text); }
}

Naming Conventions

  • Classes: LoginPage, CartPage
  • Tests: shouldLoginWithValidCredentials()
  • Locators: By loginButton, By emailInput

Common Mistakes & How to Avoid Them

  • Hard sleeps: replace with explicit waits.
  • Overusing XPath: prefer CSS/IDs where possible.
  • Not cleaning state: ensure quit() and data resets.

Common Issues & Solutions

Fix frequent exceptions with targeted strategies.

ElementClickInterceptedException

  • Cause: element covered by overlay/sticky header.
  • Fix: scroll into view or wait for overlay to disappear.

StaleElementReferenceException

  • Cause: DOM updated, reference is no longer valid.
  • Fix: re‚Äëlocate the element after page changes or waits.

NoSuchElementException

  • Cause: locator not found at the moment of search.
  • Fix: check locator, add waits, ensure visibility/DOM presence.

TimeoutException

  • Cause: waits exceeded before condition met.
  • Fix: review conditions, increase timeout, stabilize app state.

Browser Compatibility Issues

  • Keep browsers/drivers aligned in versions.
  • Use vendor‚Äëspecific options for consistent behavior.

API Testing with Selenium? (Clarification & Alternatives)

Selenium is for browser automation. For API testing, use dedicated tools like REST Assured, Postman, or Karate. Combine UI and API layers for robust coverage.
